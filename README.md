# Cocktail Recipe Telegram Bot

Read-only Telegram bot for browsing cocktail recipes from a PostgreSQL database.

The project demonstrates a backend/data pipeline:

```text
Scraping → parsing → normalization → PostgreSQL → Telegram bot → analytics
```

---

## Status

Production beta is deployed on a VPS with systemd.

Current features:

- `/cocktail <name>` — show recipe by exact cocktail name
- `/list` — browse cocktails with pagination
- `/search <query>` — search by cocktail name or ingredient
- Inline button navigation
- Cocktail cards with ingredients, image and source URL
- Text fallback for cocktails without images
- Legacy add/edit/delete flows disabled
- Minimal analytics foundation in progress

---

## Dataset

The database is generated by a separate ETL pipeline.

```text
Cocktails:              6614
Ingredients:            30761

parse_status ok:        5619
parse_status partial:   995
parse_status failed:    0

Ingredient parse coverage: ~99.96%

Real image URLs:        6360
No image:               254
Bad image URLs:         0
```

Each cocktail card includes a source URL back to the original recipe page.

## Commands

```text

/list       Browse cocktails

```

```text

/edit

```

## Analytics

Minimal analytics foundation is implemented through:

```text
analytics.bot_events
repositories/analytics_repository.py
services/analytics_service.py
```

Analytics logging is best-effort: if analytics fails, the bot continues working.

Currently tracked / planned:

- search events
- not found searches
- cocktail views
- pagination callbacks
- image fallback
- errors
- duration_ms
- top search queries
- top opened cocktails
- daily unique users

## Installation

```bash
git clone <repo-url>
cd cocktail_manager_bot_tg

python -m venv .venv
source .venv/bin/activate

python -m pip install -r requirements.txt
```

For development:

```bash
python -m pip install -r requirements-dev.txt
```

---

## Environment Variables

Create `.env`:

```env
BOT_TOKEN=your_telegram_bot_token

DB_HOST=localhost
DB_PORT=5432
DB_NAME=cocktail_etl
DB_USER=postgres
DB_PASSWORD=your_password
```

`.env` must not be committed.

## Run Locally
